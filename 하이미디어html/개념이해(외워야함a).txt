*block vs inlline (css박스모델)
- (대부분위) 요소는 블록과 인라인으로 구별된다.
- 블록요소는 가로 한 줄을 다 차지
→ 부모의 가로 크기 100%를 상속(기본값)
→ 세로 크기는 자손의 합(블록/인라인 공통)(세로 크기는 자손이 결정함)

- 인라인요소는 여유가 있을 경우, 옆자리를 다른 요소에 양보
- 오직! 블록만 크기와 여백을 가진다.
- padding, border는 크기에 영향을 준다.(크기를 늘린다)
→ 무조건 width, height로만 그리게 하려면 bxbb

- 블록요소의 가운데 정렬은, width 부여 후 margin: 0 auto;
- 블록이 감싸는 인라인 요소, 텍스트를 가운데로 보내려면 text-align: center(tac)

* 플렉스 박스
- 배치하려는 요소의 부모에게 display: flex 적용
- 인라인요소처럼 가로가 쪼그라 든다. (하지만 여전히 블록)
- 세로크기는 부모 공간(flex)의 세로 크기 기준 100% →stretch

- 부모 공간이 flex가 되면 메인축이 가로로 생겨남
- 아이템은 그 메인축으로 자동정렬


*position
- static, relative, absolute, fixed, sticky
- 모든 요소는 position: static ←기본값
- position: absolute를 적용하면 원래 자기 위치를 기준으로 3차원으로 뜬다.
→ 원하는 좌표값을 설정해서 쓰자! (top, right, bottom, left)
→ 이때, 직계 조상에 기준점 역할을 하는 세팅이 없으면 뷰포트를 기준으로 한다.
→ 기준점 역할 세팅의 가장 대표적인 옵션은 posr
→ 인라인요소처럼 가로가 쪼그라 든다. (하지만 여전히 블록)

- 절대 배치된 요소는 z-index가 기본값0으로 설정, 이 숫자가 높을수록 먼저 보인다.

- position: fixed
→ posf는 무조건 뷰포트만 기준으로 배치됨
→ 스크롤과 연동되지 않고 고정됨
